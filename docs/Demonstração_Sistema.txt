Solicitação para implementação de acesso de teste (Modo Demonstração)

Criar um modo demonstração no sistema para permitir que novos usuários conheçam o sistema antes de contratar um plano.

O modo demonstração deve permitir apenas navegação e visualização, sem permitir inclusão, alteração ou exclusão de dados.

---

OBJETIVO

Permitir que usuários testem o sistema de forma segura, sem impactar dados reais e sem consumir recursos pagos.

---

TIPOS DE USUÁRIO DEMO

Criar contas de demonstração para:

* Aluno
* Profissional
* Clínica
* Administrador

Cada perfil deve ter dados fictícios e simulados.

---

REGRAS DO MODO DEMONSTRAÇÃO

Permitir:

* Navegação no sistema
* Visualização de telas
* Visualização de relatórios
* Acesso aos dashboards
* Teste de funcionalidades em modo simulado

Bloquear:

* Criar registros
* Editar registros
* Excluir registros
* Enviar pagamentos
* Comprar créditos
* Consumir créditos reais de IA
* Alterar configurações

---

IMPLEMENTAÇÃO TÉCNICA

Adicionar campo no usuário:

is_demo = true

Criar middleware:

CheckDemoMode

Regras:

Se usuário for demo:

Bloquear métodos:

POST
PUT
DELETE

Permitir:

GET

---

MENSAGEM PADRÃO

Quando o usuário tentar salvar:

Mostrar:

"Você está em modo demonstração. Para utilizar esta funcionalidade, crie uma conta ou contrate um plano."

---

IDENTIFICAÇÃO VISUAL

Mostrar banner no topo do sistema:

Modo Demonstração

Mensagem:

"Você está utilizando o sistema em modo demonstração. Os dados não serão salvos."

---

SEGURANÇA

Garantir que o usuário demo:

Não possa:

* alterar banco de dados
* executar pagamentos
* consumir créditos de IA
* disparar webhooks
* alterar configurações

---

RESET AUTOMÁTICO

Os dados do usuário demo devem ser restaurados automaticamente:

* ao sair do sistema
  ou
* a cada 24 horas

---

BOTÃO DE CONVERSÃO

Adicionar botão:

Assinar agora

Ou:

Contratar plano

Redirecionar para:

/plano

---

COMPATIBILIDADE

O modo demonstração deve funcionar com:

* planos
* assinaturas
* créditos de IA
* relatórios
* financeiro
* dashboards
* painel do aluno
* painel do profissional
* painel da clínica

Sem impactar o sistema real.
