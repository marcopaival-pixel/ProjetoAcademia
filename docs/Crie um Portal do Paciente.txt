Crie um Portal do Paciente em um sistema SaaS de saúde e performance, com arquitetura profissional, escalável e segura.

O portal deve funcionar exclusivamente em modo somente leitura (read-only), servindo apenas para consulta de informações fornecidas pelo profissional.

🚀 CONCEITO DO SISTEMA

O Portal do Paciente é uma plataforma de acompanhamento contínuo de:

Saúde
Treinamento físico
Nutrição
Evolução corporal
Adesão comportamental

O paciente NÃO interage com edição de dados — apenas visualiza informações.

🔐 1. ARQUITETURA DE ACESSO
🔑 Modelo híbrido de autenticação
Camada 1 — Token de acesso (entrada inicial)
Gerado automaticamente no cadastro do paciente
Usado em link seguro:
/patient/access?token=xxx
Deve ser:
único
criptografado (hash no banco)
com expiração configurável
invalidado após uso (opcional)
Camada 2 — Sessão autenticada

Após primeiro acesso:

paciente cria senha OU entra automaticamente
sessão persistente segura
Camada 3 — Escopo de dados

O paciente só pode acessar seus próprios dados:

patient_id = session.patient_id
🔐 2. CONTROLE DE PERMISSÃO (RBAC + ABAC)
Roles do sistema:
admin
professional
reception
patient_viewer
Regras do paciente:
Apenas leitura (read_only)
Sem permissões de escrita
Escopo restrito ao próprio paciente
Regras adicionais (ABAC):
paciente ativo
vínculo com profissional obrigatório
acesso válido dentro de assinatura/plano (se SaaS)
🚫 3. RESTRIÇÕES OBRIGATÓRIAS

O sistema deve garantir:

❌ Nenhum POST / PUT / DELETE no portal do paciente
❌ Nenhuma edição de dados via interface
❌ Nenhuma alteração via URL ou requisição manual
❌ Backend deve bloquear alterações mesmo forçadas
❌ Interface sem botões administrativos ou de edição
🧱 4. SEGURANÇA AVANÇADA
Token armazenado como hash
Expiração obrigatória de tokens
Sessão com timeout automático
Rate limit no acesso ao portal
Logs de auditoria de acesso
HTTPS obrigatório
Validação de sessão em cada requisição
