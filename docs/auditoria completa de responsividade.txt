Realizar auditoria completa de responsividade do sistema.

ANTES DE INICIAR:
1. Analisar todas as telas do sistema
2. NÃO modificar nada inicialmente
3. Apenas identificar problemas

========================================
TESTES DE RESPONSIVIDADE
========================================

Simular os seguintes dispositivos:

1. Mobile:
- largura: 320px a 480px

2. Tablet:
- largura: 768px a 1024px

3. Desktop:
- acima de 1200px

========================================
VERIFICAÇÕES
========================================

Para cada tela:

- elementos cortados
- overflow horizontal
- botões pequenos ou difíceis de clicar
- textos ilegíveis
- menus quebrados
- formulários desalinhados
- tabelas não responsivas
- modais fora da tela

========================================
RELATÓRIO
========================================

Listar por tela:

- Nome da tela
- Problema encontrado
- Dispositivo afetado (mobile/tablet/desktop)
- Gravidade (baixa/média/alta)

========================================
CORREÇÃO (APÓS ANÁLISE)
========================================

Depois do relatório:

1. Corrigir problemas usando:
- CSS responsivo
- media queries
- flexbox/grid

2. Garantir:
- layout fluido
- boa experiência mobile-first

3. NÃO quebrar:
- funcionalidades existentes
- lógica do sistema

========================================
VALIDAÇÃO FINAL
========================================

- Re-testar todas as telas
- Confirmar que funciona em:
  - celular
  - tablet
  - desktop

========================================
OBJETIVO
========================================

Garantir que o sistema funcione corretamente em qualquer dispositivo com layout responsivo e profissional.
