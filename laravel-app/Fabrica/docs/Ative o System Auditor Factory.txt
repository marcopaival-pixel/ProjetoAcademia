Implemente no sistema NexShape a ativação e configuração do System Audit Orchestrator em produção.

Objetivo:
Permitir que o sistema execute auditorias completas automaticamente ou sob demanda, analisando código, arquitetura, segurança, performance e qualidade geral do sistema.

Requisitos:

1. Criar mecanismo de ativação do System Audit Orchestrator com 3 modos:

   a) Manual:
   - Criar endpoint: POST /api/ai/audit
   - Permitir execução via botão no painel administrativo "Executar Auditoria Completa"

   b) Automático por eventos:
   - Executar auditoria após deploy em produção
   - Executar auditoria em falhas críticas ou erros recorrentes
   - Executar auditoria quando houver mudanças grandes no código

   c) Agendado:
   - Permitir agendamento via cron job
   - Exemplo: auditoria semanal e mensal

2. Sempre que ativado, o sistema deve chamar o:
   - System Audit Orchestrator

3. O System Audit Orchestrator deve executar os seguintes agentes:
   - Code Quality Agent
   - Architecture Audit Agent
   - Performance Audit Agent
   - Security Audit Agent
   - Feature Overload Agent
   - Duplication Detector Agent
   - Test Coverage Agent

4. O Orchestrator deve:
   - Consolidar todos os resultados
   - Classificar problemas em: críticos, médios e leves
   - Gerar relatório estruturado
   - Sugerir plano de melhoria em etapas
   - Registrar logs da auditoria

5. Criar painel administrativo contendo:
   - Botão "Executar Auditoria Agora"
   - Histórico de auditorias
   - Status da última auditoria
   - Score geral do sistema (0 a 100)
   - Lista de problemas encontrados
   - Sugestões de melhorias

6. Criar estrutura de logs:
   - auditoria_id
   - tipo (manual, automático, agendado)
   - data/hora
   - agentes executados
   - problemas encontrados
   - impacto no sistema

7. Garantir que a auditoria:
   - Não altere código diretamente
   - Apenas analise e sugira melhorias
   - Seja segura para produção

8. Antes de implementar:
   - Verificar arquitetura atual
   - Reutilizar serviços existentes
   - Apresentar plano de implementação
   - Aguardar aprovação antes de alterar código
